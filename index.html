Contextualização

  • Nomes são usados para identificar entidades em um SD (processos, recursos, etc).
  • Para interagir com uma entidade, é preciso resolver o seu nome.
  • Para isso, usa-se um sistema de nomeação.

Nomes, Identificadores e Endereços

  • Um nome em um SD é uma cadeia de bits ou caracteres usada para referenciar uma entidade.
  • Para acessar uma entidade, precisamos de um ponto de acesso, que é um tipo especial de entidade.
  • O nome de um ponto de acesso é denominado endereço.
  • Uma entidade pode fornecer vários pontos de acesso.
    Ex: Uma pessoa e vários telefones.
  • Portanto, um endereço é apenas um tipo especial de nome.
  • Um identificador é um nome que tem as seguintes propriedades:
    • Referencia, no máximo, uma entidade;
    • Cada entidade é referenciada por, no máximo, um identificador;
    • Um identificador sempre referencia a mesma entidade, isto é, nunca é realizado.
  • Usando identificadores é possível referenciar uma entidade sem ambiguidade.

Nomeação Simples

  • Em alguns casos, identificadores e nomes são simplesmente cadeias aleatórias de bits às quais nos referimos, por conveniência, como nomes não estruturados, ou nomes simples.

Soluções simples para localizar uma entidade:

  • Broadcasting e Multicasting
    • Em SD’s em rede, uma mensagem contendo o identificador é enviada para todas as máquinas da rede (broadcast).
    • Somente as máquinas que possuem um ponto de acesso para a entidade enviam uma mensagem de resposta contendo o endereço do ponto de acesso.

Soluções simples para localizar uma entidade:

  • Broadcasting e Multicasting
    • Broadcasting se torna ineficiente quando a rede se torna muito grande.
    • Por isso, o Multicasting pode ser usado para localizar entidades dentro de um grupo.

Exemplo: Sistema de Nomes de Domínio (DNS)

  • Um dos maiores serviços de nomes distribuídos de nomeação em uso hoje é o Domain Name System (DNS) da Internet.
  • O DNS é usado para consultar endereços IP de servidores.
  • Espaço de nomes DNS:
    • O espaço de nomes do DNS é organizado hierarquicamente como uma árvore com raiz
    • Uma subárvore é denominada domínio: um nome de caminho até seu nó raiz é denominado nome de domínio.
